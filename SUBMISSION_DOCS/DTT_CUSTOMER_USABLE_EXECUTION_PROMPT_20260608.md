# ConceptMaster DTT 목표추진 프롬프트

글자 수: 공백 포함 2153자

```text
[목표]
ConceptMaster CodeFair를 뼈대가 아니라 Codex/Antigravity에서 바로 이어 개발·시연 가능한 고객 사용 후보 제품으로 만든다. 현재 제품은 초6 수학 오답을 오개념 데이터로 정리하고, Manus API 또는 절약형 fallback으로 AI 진단, 같은 개념 재도전, 개선률 확인, 30초 심사용 데모를 제공한다. 개발자/권리자에게 직접 코드 사용 허가는 이미 물리적으로 받았으므로 허가 재확인은 하지 말고, 엔지니어링 실행 상태만 보고한다.

[완성 그림]
사용자는 zip 또는 repo를 열고 `npm test`, `.env` 설정, `npm start`만으로 로컬 실행한다. 첫 화면은 듀오링고식 학습 홈이고, 메뉴는 `학습 / 30초 데모 / 데이터 / 연구근거`로 분리된다. 심사위원은 30초 안에 오답 기록→AI 진단→같은 개념 재도전→개선률 변화를 본다. 건호는 CodeFair 발표에서 “AI가 문제를 대신 푼 것”이 아니라 “내 오답 데이터를 분석해 같은 개념을 다시 풀게 해 주는 학습 시스템”이라고 설명할 수 있어야 한다.

[DTT 원칙]
DTT는 Definition-Test-Trace다. 기능을 추가하기 전에 정의를 한 문장으로 고정하고, 그 정의를 확인할 테스트/스모크를 먼저 정한 뒤, 통과 증거 파일 경로를 남긴다. TDD는 위험한 코드 변경 단위에만 얇게 적용한다. 작은 테스트만 반복하며 전체 목표가 흐려지면 즉시 멈춘다.

[마일스톤]
M0 현재 상태 고정: 기존 40개 테스트, 메뉴형 UI, Manus 절약 정책, 제출 문서 상태를 읽고 baseline 리포트를 만든다. Done: `npm test` PASS와 현재 URL/스크린샷 증거.
M1 고객 실행 하네스: `AGENTS.md`, `CLAUDE.md`, `docs/PRD.md`, `docs/ARCHITECTURE.md`, `docs/WORK_LANES.md`, `phases/CURRENT_PHASE.md`, `scripts/execute.py validate`를 만든다. Done: Codex/Antigravity가 같은 명령으로 다음 일을 이어 할 수 있음.
M2 API/환경 안정화: `.env.example`, 서버 전용 `MANUS_API_KEY`, credit saver, live 1회 버튼, fallback 상태 표시를 정리한다. Done: 키가 브라우저에 노출되지 않고 API 없음/느림/실패에도 데모 가능.
M3 학습 제품 완성: 오답 기록, AI 진단 카드, 같은 개념 재도전, 복습 스케줄, 개선률, 데이터 탭을 학생이 실제로 이해할 수 있게 연결한다. Done: 한 번 오답을 내면 화면 증거가 모두 갱신됨.
M4 심사 패키지: 30초 데모, 2분 발표 대본, 예상 질문답변, 작품요약서/설명서, GitHub 참고 근거를 최신 UI와 맞춘다. Done: 건호가 그대로 연습 가능.
M5 고객/건호 인수: README, HANDOFF, 실행 체크리스트, 문제 추가 방법, AI 비용 절약 규칙, 알려진 한계를 정리한다. Done: 다른 개발자가 repo를 열고 15분 안에 실행/수정 가능.
M6 인간 검증: 건호가 직접 1회 실행하고 막힌 지점, 설명 가능한 부분, 수정 필요 부분을 기록한다. Done: Human-Verified는 실제 사용 후에만 표시.

[실행 순서]
1. 먼저 현재 파일과 테스트를 읽고 baseline을 만든다.
2. 하네스 파일을 만들고 `scripts/execute.py validate` 하나로 검증을 묶는다.
3. M1부터 M6까지 한 번에 모두 구현하지 말고, 각 M마다 Definition-Test-Trace를 닫은 뒤 다음으로 간다.
4. UI 변경은 브라우저 스크린샷으로 확인하고, API 변경은 키 노출/절약/fallback 테스트를 통과해야 한다.
5. 최종 답변은 변경 파일, PASS/FAIL, 증거 경로, 다음 한 단계만 보고한다.

[중단 규칙]
같은 오류/명령/브라우저 경로가 5회 반복 실패하면 즉시 중지하고, 현재 PASS 증거와 정확한 blocker만 보고한다. GUI/브라우저 타임아웃은 같은 경로 2회 실패 시 중지한다. 새 증거 없이 GitHub 재검색, 문서 재작성, 미세 테스트 반복만 계속되면 토큰 낭비로 판단하고 멈춘다. 고객 사용 가능, Human-Verified, 배포 가능은 실제 검증 전까지 주장하지 않는다.
```
