# 듀오링고 감성 이미지/화면 프롬프트

## 범위 판단

제출 핵심은 "초6 수학 오답 DNA 지도"로 유지한다. 과학은 화면에서 "다음 확장 과목" 또는 "준비중"으로 보여주는 정도가 좋다.

이유:

- 수학과 과학을 둘 다 본 기능처럼 쓰면 문제은행, 오개념 분류, 테스트, Human-Verified가 두 배로 늘어난다.
- 코드페어 심사에서는 좁고 완성도 높은 작품이 넓고 덜 검증된 작품보다 강하다.
- 과학 탭은 확장 가능성을 보여주는 장치로만 쓰면 오히려 좋아진다.

추천 표기:

- 메인 제목: 오답 DNA 지도
- 부제: 초6 수학 AI 오답 코치
- 과목 탭: 수학 / 과학 준비중
- 확장 문장: 과학도 같은 방식으로 확장할 수 있도록 설계했습니다.

## 이미지 생성 프롬프트

Duolingo-inspired but fully original Korean educational app illustration, no Duolingo logo, no owl mascot, no copied characters. Create a friendly mobile-first learning dashboard for an elementary student AI wrong-answer coach called "오답 DNA 지도". The screen should feel playful, bright, rounded, and motivating like a premium language-learning app, but focused on Korean grade-6 math.

Main visual: an original cute student helper mascot holding a tablet, standing beside a colorful "wrong-answer DNA path". The path has five clear learning nodes: "오답 기록", "AI 진단", "같은 개념 재도전", "개선률 확인", "오늘 복습". Use simple Korean labels, large readable typography, soft green as the primary success color, with small accents of blue, yellow, and coral. Add a subject switcher at the top: "수학" active, "과학 준비중" locked with a small beaker icon. Do not show science as fully implemented.

Include one main card: "오늘의 오답 개념: 분수의 통분". Add a small progress bar labeled "이해 회복 50%". Add a tiny data card with friendly labels "풀이 기록표" and "개념 요약표" instead of technical terms. The visual should communicate that the student learns from mistakes, not that AI gives all answers.

Style: clean vector illustration, rounded UI, warm white background, high contrast Korean text, child-friendly but not childish, minimal clutter, no dense analytics dashboard, no dark theme, no purple-blue gradient dominance, no stock photo look. First impression should be: "friendly AI learning coach that helps me understand why I made the same mistake."

## 프론트엔드 디자이너/개발자 프롬프트

You are designing a Duolingo-quality but original Korean CodeFair demo interface for ConceptMaster, "오답 DNA 지도". Do not copy Duolingo assets, logo, owl, exact colors, or UI. Use the learning-product principles: playful progression, clear daily goal, one primary action, friendly mascot, compact feedback, and visible progress.

Redesign the current dashboard around one vertical learning journey instead of many equal cards. The first screen must immediately show:

1. Product identity: "오답 DNA 지도"
2. Scope: "초6 수학 AI 오답 코치"
3. Subject switcher: "수학" active, "과학 준비중" disabled
4. Main mission: "분수의 통분과 덧셈 다시 이해하기"
5. One primary button: "심사용 데모 시작"
6. Learning path nodes: 오답 기록 -> AI 진단 -> 같은 개념 재도전 -> 개선률 확인
7. Friendly explanation bubble from an original mascot: "틀린 이유를 찾고 같은 개념을 다시 풀어보자!"
8. Compact evidence drawer or menu: 풀이 기록표, 개념 요약표, AI 진단 기록, 테스트 결과

Interaction rules:

- Keep the user on a clear path. Do not show every data card at once.
- Use tabs or bottom navigation: 오늘 학습, 오답 DNA, 데이터 증거, 제출 자료.
- Hide developer terms from the main screen. Replace `attempt_log` with "풀이 기록표", `concept_summary` with "개념 요약표", fallback with "예비 진단", `needs_review` with "사람 확인 필요".
- Keep science as a locked future expansion, not a working claim.
- The CodeFair judge should understand within 10 seconds: this app stores wrong-answer data, AI explains the mistake, and the student retries the same concept.

Visual rules:

- Rounded but restrained cards, no nested cards.
- Primary color green, secondary blue/yellow/coral accents.
- Large readable Korean labels.
- A mascot can appear, but it must be original and not owl-like.
- Use icons for math, science, AI, retry, and progress.
- Avoid a crowded analytics dashboard on the first screen.
- The data-heavy evidence should appear after the judge clicks "데이터 증거".

Acceptance criteria:

- A judge can tell the app is for grade-6 math within 3 seconds.
- A judge can tell science is future expansion, not current verified scope.
- A judge can see the full wrong-answer recovery loop within 10 seconds.
- No English internal IDs are visible on the main learning screen.
- The interface supports the CodeFair story: AI + data + real learning problem + verified demo.
